The Problem

When your agent acts on your behalf, the infrastructure underneath determines whether you can trust how it operates.

Agents that Pay

Your agent moves money, but there's no way to scope what it can spend or verify it spent correctly.

Scope tokens enforce per-tx limits, daily caps, and address whitelists onchain.

Agents that Trust

Trust flows through centralized registries. If the provider shuts down, your agent loses access.

Onchain attestations + ERC-8004 identity. No single point of failure.
!

Agents that Cooperate

Deals are enforced by platforms. If the platform changes rules, your deal gets rewritten.

Smart contract escrow with milestones. Terms enforced by protocol, not companies.

Agents that Keep Secrets

Every API call creates metadata about you. Spending patterns, contacts, behavior -- all leaking.

Key separation + scope tokens prove authorization without revealing the human.

The Innovation: Scope Tokens

Permissions as ERC-1155 tokens. Holding the token = authorized. Burn it = instant revocation. No platform can silently change what your agent is allowed to do.

How It Works

Three roles. Zero shared secrets. Full onchain auditability.

1

Human Sets Rules

The owner mints scope tokens defining what the agent can do: spending limits, allowed contracts, deal authority, attestation rights. All encoded onchain as ERC-1155 tokens.

2

Agent Operates

The agent uses its OWN wallet (never the owner's). It can spend, make deals, build trust -- but only within the boundaries the scope tokens allow. The contract rejects anything out of scope.

3

Contract Enforces

Every action is validated onchain and produces a receipt. The human can audit everything. To revoke access: burn the scope token. Instant. Irreversible. No middleman involved.
